BETA ENGINE

Beta Engine is a game engine created to develop a Year 1 game project.

An editor interface was not required for a Year 1 project. However, we recognized the importance of an editor, and thus implemented one in the short amount of time we had.

Spellthief, a DigiPen award-winning game, was created with Beta Engine.

Here are some of the tools I developed for Beta Engine:

Animation Editor

Real-time preview of the animation nodes

The animation editor allows designers to modify animation nodes and see the animation happening in real-time. This means that they can adjust the speed with the most accurate feedback about how it looks.

The animation editor also supports adding, editing, and deleting of parameters and transitions.

Why

We had many animated art assets created for Spellthief, each with a handful of states and transitions (idle, moving, attacking, etc.). 

This improved the pipeline when it came to implementing animations, especially when trying to ensure that animations transited into other animations accurately.

Furthermore, speed adjustment was key in ensuring that the artist did not have to modify the art asset just to remove or add frames in order to make the animation look good.

Prefab Tiler

Tiling with position offsets

The prefab tiler allows designers to place down art assets as a game object that comes with a specified collider (or none at all). The position and rotation of the game object can be specified in the tiler.

One QoL feature of the tiler is the position offset feature, which automatically modifies the position of the next tile to improve productivity in placing down seamless tiles.

Why

Spellthief's levels are made out of tilesets, and it takes a lot of time to copy, paste, and adjust the positions of the tiles. The prefab tiler improved the time taken to set up a level by a lot, and ensured that there were minimal errors in the positions of the tiles.

Dialogue Editor

Dialogue Editor interface

The dialogue editor allows designers to implement dialogue in an easy-to-use fashion.

They can set the speaker's name, offset of the dialogue box, whether the dialogue stops at that point, stops input from being processed, and the VFX that plays with it.

If they set the dialogue to be a breakpoint, the next line can be easily called to continue the dialogue from code.

Why

Implementing dialogue is a nightmare for most designers; setting up speech boxes and programming the flow is a huge hassle, especially without good programming practices. 

By providing an interface for dialogue to be implemented, storytelling in games is no longer a huge time sink.
