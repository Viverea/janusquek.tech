FORGE ENGINE

A game engine created to develop a Year 2 and Year 3 game project, Forge Engine has various improvements over its predecessor, Beta Engine.

Howl: A Tale of Wolves, a DigiPen award-winning game, was created with Forge Engine.

Here are some of the technologies I developed for Forge Engine:

C# Scripting API & Integration

Just like in Unity

One feature that Forge Engine has that stands out over Beta Engine is C# scripting support. It follows Unity's C# scripting API for the most part when it comes to function names. keywords and behaviors (Start, Update, OnDestroy). 

Many features seen in Unity C# scripting are supported, including OnCollision/Triggers, co-routines, and getting of other components/scripts.

Prefabs

Here, we see apply and unlink in action

Forge Engine's prefab system supports most of what Unity's prefabs would support: applying, reverting, and unlinking. Linking is just a fun little function that links a transform to a prefab, essentially replacing it with an instance of the prefab.

Memory Management

By overriding new and delete, all instances of memory allocation is managed by the memory manager

Forge Engine features its own in-house memory manager that manages memory across all allocation calls by overriding the new operator. 

The memory manager works by malloc-ing a huge pool of memory at the start of the process and splitting it up into nodes of 16-byte aligned memory.

This provides a performance boost as we reduce the amount of system calls and provide contiguous memory in certain cases.

Adaptive Audio

Forge Engine has an adaptive audio system, dubbed MAGES (Multifaceted Audio for Games and Entertainment System). It is capable of setting up modes, layers, and nodes that allows designers to layer multiple sounds on top of each other, each with their own durations and looping behavior.

MAGES also supports transitions between modes and playing of standalone audio clips based on BPM: this allows for adaptive audio to be played seamlessly while following the beat.

File I/O API

No file paths, just a system name, property name and a value

Forge Engine has a configurations manager that can easily get and set properties without the users having to deal with file paths. They just have to state whether is it per project, user, project x user, or app data.

The configurations manager supports primitive types such as integers, float, booleans, and strings, as well as structured objects and arrays of primitive types and structured objects.

Lockless Unordered Map

While it is not a core part of Forge Engine, the lockless unordered map is a container made to work in multithreaded environments. It supports inserting and erasing across multiple threads without any locking operations.

It is currently used to store physics and networking data, where it is used to process data using multiple threads (i.e. checking for collisions).
